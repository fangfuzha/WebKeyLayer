## 📋 协议规范详情（新增）

### API_PROTOCOL.md 包含内容

**1. WebSocket 推流协议**

```
消息格式: {id, type, timestamp, payload}
6 种消息类型:
- connection_established (初始化)
- key_pressed (按键按下)
- key_released (按键松开)
- config_updated (配置变更)
- heartbeat (心跳)
- disconnect (连接关闭)
```

**2. HTTP 管理 API**

```
11 个端点:
- GET /api/config
- POST /api/config
- POST /api/config/reload
- GET /api/preset/list
- POST /api/preset/import
- GET /api/preview
- GET /api/logs
- GET /api/status
- POST /api/service/start
- POST /api/service/stop
- GET /api/network/ip
```

**3. 性能指标**

```
延迟目标:
- 按键端到端 < 30ms
- WebSocket 往返 < 20ms
- 心跳响应 < 5ms

吞吐量目标:
- 并发客户端 8+
- 事件处理 1000/秒
- 带宽占用 0.5-50 KB/s
```

**4. 版本规划**

```
v1.0 (当前):
- 字母/数字/方向键/WASD
- 第一阶段预设元素 (keyboard, mouse, texture)

v1.1+ (规划):
- 鼠标事件
- 手柄事件
- 第二阶段预设类型
- 消息压缩
- 二进制格式
- 公网穿透
```
